# ИНСТРУКЦИЯ ПО ИСПОЛЬЗОВАНИЮ
# UNICON DLP/DPI Pilot 1.0.3

---

## 1. ДОСТУП К СИСТЕМЕ

**Web Console:** `http://45.150.25.74/`

В боковой панели — 11 вкладок:
- **Overview** — сводка, загрузка файлов на анализ
- **Incidents** — инциденты безопасности
- **Events** — все события
- **Agents** — компьютеры с агентами
- **Policies** — DLP-политики
- **Notifications** — оповещения
- **Audit** — журнал аудита
- **Operations** — операционная сводка
- **Handoff** — документация пилота
- **Demo** — демо-режим
- **Scenarios** — сценарии тестирования

Статус API (вверху справа): **API Online** = система работает.

---

## 2. МОНИТОРИНГ КОМПЬЮТЕРОВ (АГЕНТЫ)

### Установка агента на Windows-ПК:

1. Скачать: `http://45.150.25.74/dlpdpi-agent-v6.tar.gz`
2. Распаковать
3. Запустить `setup.bat` от имени Администратора

### Что отслеживает агент:

| Тип события | Описание |
|-------------|----------|
| `endpoint.file.risky_extension` | Подозрительный файл (.exe, .bat, .ps1, .dll...) |
| `endpoint.file.removable_copy` | Копирование на USB-флешку |
| `endpoint.file.sensitive_detected` | Файл с ключевыми словами (договор, ПИНФЛ...) |
| `endpoint.usb.blocked` | Блокировка USB-устройства |

### Просмотр агентов:

**Agents** → список компьютеров:
- 🟢 `fresh` — онлайн, heartbeat в норме
- 🟡 `stale` — нет связи >5 минут
- 🔴 `offline` — нет связи

## 3. DLP-АНАЛИЗ ФАЙЛОВ

### Загрузка файла на проверку:

**Overview** → кнопка **Analyze File** (или **Sample PINFL** для теста)

Поддерживаемые форматы: `.txt`, `.doc`, `.docx`, `.pdf`, `.xls`, `.xlsx`, `.csv`

### Что ищет система:

| Политика | Тип данных | Серьёзность |
|----------|-----------|-------------|
| PINFL | Персональный идентификатор (14 цифр) | CRITICAL |
| Passport / ID | Паспортные данные | CRITICAL |
| Bank Card | Номер банковской карты | CRITICAL |
| INN | ИНН (12 цифр) | HIGH |
| Email / Phone | Контакты | MEDIUM |
| File Risk | Вредоносные индикаторы | HIGH |

### Результат анализа:

После загрузки → вкладка **Incidents** → новый инцидент с деталями:
- Что найдено (тип данных)
- В каком файле
- Серьёзность
- Рекомендуемое действие

---

## 4. РАБОТА С ИНЦИДЕНТАМИ

### Просмотр:

**Incidents** → список всех инцидентов. Фильтры:
- **Status** — New, In Progress, Resolved...
- **Severity** — Critical, High, Medium, Low
- **Source** — Upload, Endpoint, DPI

### Обработка инцидента:

1. Нажать на строку инцидента
2. Изучить детали (что, где, когда)
3. Сменить статус: New → Assigned → In Progress → Resolved → Closed
4. Или пометить как **False Positive** (ложное срабатывание)

---

## 5. СОБЫТИЯ (EVENTS)

**Events** → все события в реальном времени.

Колонки таблицы:
- **Event** — тип события
- **Source** — источник (endpoint/dpi/upload)
- **Severity** — серьёзность
- **Action** — действие (alert/block/quarantine)
- **Asset** — компьютер/хост
- **Path/File** — путь к файлу (наведите для полного пути)
- **Created** — время

Фильтры: Source, Severity, Action, Hostname, Application.

---

## 6. DPI-МОНИТОРИНГ СЕТИ

Система анализирует сетевой трафик через DPI-сенсоры:

| Тип угрозы | Описание |
|-----------|----------|
| `tor_exit_node` | TOR-выходной узел |
| `commercial_vpn` | Коммерческий VPN |
| `botnet_c2` | Ботнет/C2-сервер |
| `crypto_mining` | Криптомайнинг |
| `dns_tunnel` | DNS-туннель |
| `cloud_storage` | Облачное хранилище |

События DPI отображаются в **Events** (source=dpi) и **Incidents**.

---

## 7. ПОЛИТИКИ (POLICIES)

**Policies** → управление правилами DLP.

7 предустановленных политик:
- PINFL, Passport, Bank Card, INN, Contacts, File Risk, Endpoint Monitor

Каждую политику можно:
- Включить/выключить (Enabled)
- Изменить действие: **Alert** / **Quarantine** / **Block**
- Изменить серьёзность

---

## 8. УВЕДОМЛЕНИЯ (NOTIFICATIONS)

Система отправляет оповещения при инцидентах с серьёзностью **High** или **Critical**.

Просмотр: **Notifications** → список отправленных/ожидающих.

---

## 9. АУДИТ (AUDIT)

**Audit** → журнал всех действий в системе.

- **Verify Chain** — проверка целостности цепочки аудита
- **Export CSV** — выгрузка для внешнего анализа

## 10. ОПЕРАЦИИ (OPERATIONS)

**Operations** → сводная панель для операторов SOC:

- Количество инцидентов по статусам и серьёзности
- Количество событий по источникам
- Статус агентов (online/offline)
- Статистика уведомлений
- Проверка целостности аудита

## 11. РЕШЕНИЕ ПРОБЛЕМ

### "API unavailable" в статусе:
1. Нажать кнопку **Refresh** в боковой панели
2. Или обновить страницу (F5)
3. Проверить сервер: `curl http://45.150.25.74:80/health`

### Агент не появляется в Agents:
1. Проверить, запущена ли задача в Task Scheduler
2. Переустановить агента: `uninstall.bat` → `setup.bat`
3. Проверить соединение: `ping 45.150.25.74`

### Не загружается файл на анализ:
1. Проверить размер файла (макс. 25 МБ)
2. Проверить формат (txt, doc, docx, pdf, xls, xlsx, csv)
3. Нажать F12 → Console → показать ошибки

## 12. БЫСТРЫЕ КОМАНДЫ (PowerShell)

```powershell
# Проверка API
Invoke-RestMethod http://45.150.25.74/health

# Проверка инцидентов
Invoke-RestMethod http://45.150.25.74/api/v1/incidents

# Проверка агентов
Invoke-RestMethod http://45.150.25.74/api/v1/agents

# Загрузка файла на анализ
Invoke-RestMethod -Uri http://45.150.25.74/api/v1/objects -Method POST -Form @{file=Get-Item "test.docx"}
```
